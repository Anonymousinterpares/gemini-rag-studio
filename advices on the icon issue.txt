There are several common causes for this. Here's a checklist:

1. fill: currentColor + inherited color is black
This is the most common cause. If your SVG uses currentColor, it inherits from the CSS color property:

CSS

/* ❌ Arrow and circle both inherit black */
.scroll-button svg {
  color: black; /* or no color set, defaults to black */
}

/* ✅ Fix: set the color explicitly */
.scroll-button svg {
  color: white; /* or whatever you need */
}
2. Both circle and arrow have the same fill
The arrow exists but is invisible because it's the same color as the circle:

HTML

<!-- ❌ Both elements are black -->
<svg viewBox="0 0 24 24">
  <circle cx="12" cy="12" r="12" fill="black"/>
  <path d="M12 16l-4-4h8z" fill="black"/>  <!-- invisible! -->
</svg>

<!-- ✅ Fix: give them different fills -->
<svg viewBox="0 0 24 24">
  <circle cx="12" cy="12" r="12" fill="#4A90D9"/>
  <path d="M12 16l-4-4h8z" fill="white"/>
</svg>
3. CSS blanket rule overriding all SVG fills
Check your stylesheets for aggressive rules:

CSS

/* ❌ This kills all SVG colors */
svg * {
  fill: black;
}
/* or */
svg path, svg circle {
  fill: currentColor;
}

/* ✅ Fix: scope it or override for your button */
.scroll-button svg circle {
  fill: #4A90D9;
}
.scroll-button svg path {
  fill: white;
}
4. Circle is rendered AFTER the arrow (z-order)
In SVG, elements are painted in document order — later elements appear on top:

HTML

<!-- ❌ Circle covers the arrow -->
<svg viewBox="0 0 24 24">
  <path d="M12 16l-4-4h8z" fill="white"/>
  <circle cx="12" cy="12" r="12" fill="black"/>  <!-- painted last = on top -->
</svg>

<!-- ✅ Fix: circle first, arrow second -->
<svg viewBox="0 0 24 24">
  <circle cx="12" cy="12" r="12" fill="#4A90D9"/>
  <path d="M12 16l-4-4h8z" fill="white"/>
</svg>
5. Electron / Chromium CSP stripping inline styles
If you set colors via the style attribute and have a strict Content Security Policy:

HTML

<!-- ❌ May be stripped by CSP -->
<svg>
  <circle style="fill: blue" .../>
</svg>

<!-- ✅ Use presentation attributes instead -->
<svg>
  <circle fill="blue" .../>
</svg>
Quick Debugging Steps
Right-click → Inspect the SVG in Electron DevTools (Ctrl+Shift+I)
Look at the Computed styles for each SVG child element
Check what fill resolves to on both the circle and the arrow
Look for any CSS rule with higher specificity overriding your colors

TL;DR: In 90% of cases it's either a CSS rule setting fill on all SVG elements to black/currentColor, or the circle is painted after (on top of) the arrow in the SVG markup. Inspect the element in DevTools to confirm which one.

Your floating “scroll-to-bottom” button is showing a plain black circle with no visible arrow because the SVG you are using is almost certainly one of the very common problematic ones that developers copy/paste from the internet (usually Heroicons solid, Lucide, Remix Icon, Tabler, or a random CodePen/ChatGPT example).

These are the real reasons this happens 99% of the time:

The SVG uses fill="currentColor" (or no fill → defaults to black) for BOTH the circle and the arrow → everything becomes the same color → arrow disappears into the circle.
The circle is drawn after the arrow in the SVG code → the black/opaque circle is rendered on top and completely covers the arrow.
The SVG was exported/designed for dark mode or as a mask → it is literally just black shapes on transparent background, so on light chat background it looks like a black blob.
Some global CSS or Tailwind config forces fill: black or color: black on all svg elements.
Quickest 100% reliable fix (the way every production chat app actually does it)

Stop trying to put the circle inside the SVG. Let CSS create the colored circle and only put a simple white arrow inside the SVG.

HTML

<!-- Put this wherever you render the floating button -->
<div 
  id="scroll-to-bottom-btn"
  class="fixed bottom-6 right-6 z-50 flex items-center justify-center 
         w-12 h-12 rounded-full bg-gray-800/90 hover:bg-gray-700/90 
         backdrop-blur-sm shadow-lg cursor-pointer transition-all 
         opacity-0 translate-y-4" 
         style="transition: all 0.3s ease;"
  onclick="scrollToEnd()">

  <svg xmlns="http://www.w3.org/2000/svg" class="w-6 h-6 text-white" 
       viewBox="0 0 24 24" fill="none" stroke="currentColor" 
       stroke-width="2.5" stroke-linecap="round" stroke-linejoin="round">
    <path d="M12 19l-7-7 1.4-1.4 4.6 4.6V5h2v10.2l4.6-4.6L19 12l-7 7z"/>
    <!-- This is the most commonly used down-arrow that actually looks good -->
  </svg>
</div>
And a tiny bit of JS to show/hide it + scroll:

JavaScript

function scrollToEnd() {
  const container = document.getElementById('your-chat-container-id');
  container.scrollTop = container.scrollHeight;
}

const btn = document.getElementById('scroll-to-bottom-btn');
const chatContainer = document.getElementById('your-chat-container-id');

chatContainer.addEventListener('scroll', () => {
  const isAtBottom = chatContainer.scrollHeight - chatContainer.scrollTop <= chatContainer.clientHeight + 100;
  btn.style.opacity = isAtBottom ? '0' : '1';
  btn.style.transform = isAtBottom ? 'translateY(16px)' : 'translateY(0)';
});
Why this works every time:

The circle is pure CSS → you control the exact color/transparency/shadow
The SVG only contains the arrow with text-white → impossible for it to be black or hidden
Works in Electron, Chrome, Firefox, dark mode, light mode, high DPI, everything.
If you really want to keep your current SVG (for whatever reason)

Open it in Chrome dev tools → right-click the <svg> → Edit as HTML, then:

Option A (fastest):
Add this directly on the <svg> tag:

HTML

<svg ... fill="white" style="color: #6366f1 !important;">
Option B:
Force the arrow to be visible:

CSS

#your-button svg path { fill: white !important; }
#your-button svg circle { fill: currentColor !important; }
Option C:
Reorder the elements (circle first, arrow last) or just delete the <circle> entirely and use CSS background like in the working example above.

99% of developers who fight with black-circle scroll-to-bottom icons for hours end up switching to the CSS-circle + white-arrow method and never look back. Just do that and save yourself the pain.
